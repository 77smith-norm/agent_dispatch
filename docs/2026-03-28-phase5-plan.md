### 5.6 — Test hardening
Add or improve tests for the following gaps:

**5.6a — Edge cases in `DispatchRequest` model**
- `endpoint` with trailing slash vs. no trailing slash (normalisation)
- `thread.messages` with empty content string (should pass — empty string is valid)
- `metadata` with nested dicts (should pass)
- `agent_id` with whitespace-only string (should fail `min_length=1` ... or should it? Test and document)

**5.6b — DB edge cases**
- `get_dispatch` on unknown ID raises `KeyError` (already tested — verify message text)
- Concurrent inserts to same agent_id: second one returns `WALKIE_TALKIE_VIOLATION` (property test already covers this — add a deterministic unit test too)
- `mark_replied` and `mark_failed` called twice on same record (idempotency — what should happen?)

**5.6c — Network error coverage**
- Timeout (`httpx.TimeoutException`) → `FAILED` state, exit code 1
- DNS failure (`httpx.ConnectError`) → `FAILED` state, exit code 1
- 500 server error → `FAILED` state with error message preserved in DB
- 429 → `FAILED` state, exit code 2 (already exists — add deterministic unit test)

**5.6d — CLI output contract**
- `agent_dispatch schema` output is valid JSON and matches `DispatchRequest.model_json_schema()`
- `agent_dispatch send --output json` always produces parseable JSON even on error paths
- `agent_dispatch send` with malformed `--json` input exits with code 1 and valid JSON error

**5.6e — `__main__.py` entry point**
- Smoke test: `python -m agent_dispatch --help` exits 0

```
uv run ruff check . && uv run ty && uv run pytest -v
```

All tests green. No ruff errors. No ty errors.

## Commit sequence (TCR discipline)
Each numbered step above is one commit. Do not batch multiple steps into one commit.
Commit message format: `chore: <description>` for non-feature work, `test: <description>` for test additions.
